The first Adapter read-only property we want to use is called isLoading. It is a boolean value indicating whether there are any pending processes running by the uiScroll. So when datasource.adapter.isLoading is true, it means that the uiScroll is working right now and the viewport might be updated soon. 125ms delay was added to the Datasource.get implementation in this sample.

In addition to read-only scalar isLoading property there is also isLoading$ property which is Observable. A Subject, to be exact. In this demo we have a subscription and isLoadingCounter incrementing each time the isLoading becomes false. We see that the initial value of this counter on the UI (before any scrolling) is 1. So the uiScroll does initialization and first 3 fetches in a single session.
